GSM-DC: How Irrelevant Context Distracts LLM Reasoning

TL;DR: GSM-DC is a controlled synthetic benchmark for quantifying and improving LLM robustness to irrelevant context (IC). We model problems as symbolic DAGs, inject controllable distractors, parse chain-of-thought (CoT) solutions stepwise, uncover a power-law error scaling with distractor count, and show hard-IC training + PRM-guided beam search yields the strongest robustness (up to +6.29% OOD step accuracy).

1. Motivation

Observation. A single irrelevant sentence can derail LLM math reasoning.
Gap. Prior datasets lack systematic control over distractor structure, depth, and evaluation granularity.
Goal. Provide a clean causal testbed to separate (a) path selection errors from (b) arithmetic execution errors under injected noise.

Core Questions

  1. Scaling: How does error grow with distractor count m and reasoning depth rs?
  2. Sensitivity: Can we characterize an IC susceptibility exponent (\delta(rs))?
  3. Training: Do models benefit more from clean, mixed, or hard IC curricula?
  4. Generalization: Does exposure to strong IC transfer to out-of-distribution depths?
  5. Inference-Time Mitigation: Can stepwise reward-guided search (PRM + beam/ToT) salvage OOD robustness?

Main Contributions (High-Level)

  1. Controlled Benchmark: Symbolic DAG → deterministic distractor injection → natural language realization → automatic stepwise parser.
  2. Empirical Laws: Error follows (E(m; rs) \propto m^{\delta(rs)}) with (\delta) increasing in depth (deeper chains amplify distraction).
  3. Training Findings: Hard-IC continued pretraining > mixed / clean; robustness gains persist ID & OOD; IC disrupts both path and arithmetic.
  4. Inference Upgrade: PRM-guided beam search adds up to +6.29% OOD step accuracy beyond best trained baseline.

2. Benchmark Design

2.1 Dependency Graphs

  • Each problem: DAG (G) with a solution path (P) (length = reasoning steps (rs)).
  • Node types: direct, implicit, instance dependencies.
  • Topological sort fixes canonical ordering, enabling deterministic CoT verification.

2.2 Controllable Distractor Injection

  • Inject (m) off-path nodes → augmented (G’) (acyclic).
  • Intensity tiers (Light / Medium / Hard) = quantiles of empirical distractor counts per (rs).
  • Trade-off: Larger (rs) consumes structural budget → fewer distractor slots (bounded edges (E)).

2.3 Natural Language Realization

  • Hierarchical entity vocabulary + relation templates.
  • Modulo-5 arithmetic to keep intermediate numbers bounded (prevents explosion; supports exact parsing).
  • Distractors are structurally valid but goal-irrelevant sentences, shuffled with relevant ones.

2.4 Stepwise Evaluator

Metrics (all binary per instance): | Metric | Checks | Purpose | |——–|——–|———| | SAcc | All intermediate equations correct + symbol order valid | Arithmetic + dependency execution | | PAcc | Only required nodes & dependencies used | Path focus / distraction resistance | | EAcc | Final extracted answer exact | End-task correctness |

3. Empirical Findings

3.1 Degradation Curves

  • All six tested models show monotonic SAcc & PAcc decline with (m); EAcc lags (remains higher due to rigid format adherence).

3.2 Power-Law IC Sensitivity

  • Fit (E(m; rs)) to (m^{\delta(rs)}); (\delta(rs)) rises with depth → compounding noise amplification.
  • Interpretation: Depth acts as a multiplicative difficulty amplifier for distractor filtering.

3.3 Clean vs IC vs Hard-IC Training

  • Hard-IC pretraining yields best in- and out-of-distribution robustness.
  • Mixed IC < Hard-IC ⇒ difficulty > diversity for robustness signal.
  • Ratio ( \text{SAcc}/\text{PAcc} ) lower for clean-trained model → arithmetic corruption persists after selecting correct path (IC affects execution stage).

4. Inference-Time Robustness: PRM + Beam / ToT

4.1 Process Reward Model (PRM)

  • Training labels generated by parser: first-error truncation → prefix [+], suffix [−].
  • Synthetic negative variants inject arithmetic mistakes & irrelevant symbol definitions.

4.2 Guided Search

  • Stepwise generation with beam (or ToT) keeps top-K partial traces by PRM score.
  • Gains concentrate in OOD depths: up to +6.29% SAcc improvement without harming ID performance.

4.3 Why It Works

  • PRM penalizes early divergence, steering beams away from distractor-influenced or miscomputed branches before arithmetic errors cascade.
